# 评价结果导出数据结构说明

> 本文件用于问卷回收后,按结构提取评价数据。每位评价者在工具内点"导出"会生成 **1 个 Excel(.xlsx) + 可选 1 个 JSON**,以评价者编号命名。汇总时将所有评价者的文件放在同一目录批量读取即可。

---

## 0. 文件命名与整体约定

- 文件名:`安全性评价_评价者XX.xlsx` / `安全性评价_评价者XX.json`
  - `XX` 为评价者编号,两位补零(`01`–`26`)
- **评价者编号 → 评价模式**(对评价者不可见,仅数据侧区分):
  - `01`–`13`:外部盲评(单层级安全性 + 自填问题归因)
  - `14`–`26`:内部扩展(总体 + SR1/SR2 分维度 + 维度勾选归因)
- **图像池**:外部与内部使用**两套不同的 52 张图像**(external=manifest-existing、internal=manifest-new);同一模式内 13 人共评同一套 52 张,移位-20 循环分配,每人 20 张,**每张图恰被 5 人评价**。
- 因此跨评价者聚合时:**同模式内按 `街景点位pid` 分组**,每个 pid 在该模式下有 5 份评价。

### 安全性评级取值(全文统一)
| 数值 | 文字 |
|---|---|
| 1 | 很差 |
| 2 | 较差 |
| 3 | 一般 |
| 4 | 较好 |
| 5 | 很好 |

Excel 中评级以**文字**存储;JSON 中同时给 `文字` 与 `数值`。

---

## 1. Excel 结构(主用)

每个 `.xlsx` 含**两个工作表**:

### 工作表① `评分总表` — 每图一行(人类可读概览)

每位评价者 20 行。列结构按模式不同:

#### 外部盲评(评价者 01–13)
| 列名 | 含义 | 取值 |
|---|---|---|
| 评价者编号 | 评价者 ID | 1–13 |
| 图像序号 | 该评价者分配中的顺序 | 1–20 |
| 图像文件名 | 原始文件名 | 如 `xxx_front_view.jpg` |
| 街景点位pid | 去掉 `_front_view` 后缀的点位标识 | 如 `xxx` |
| 安全性评级 | 总体评级(文字) | 很差/较差/一般/较好/很好,未评则为空 |
| 完成状态 | 是否完成该图 | 已完成 / 未完成 |
| 无明显问题 | 是否勾选"无明显问题" | 是 / (空) |
| 无明显问题说明 | 勾选时的可选说明文字 | 文本,未勾选则为空 |
| 问题数量 | 已填写的问题条目数 | 整数;勾选"无明显问题"时为 0 |
| 问题类型概览 | 各问题类型以 `、` 连接 | 如 `照明不足、视线受阻`;勾选无明显问题时为空 |
| 提交时间 | 最后操作时间 | ISO 时间戳 |

#### 内部扩展(评价者 14–26)
| 列名 | 含义 | 取值 |
|---|---|---|
| 评价者编号 | 评价者 ID | 14–26 |
| 图像序号 | 分配顺序 | 1–20 |
| 图像文件名 | 原始文件名 | 如 `xxx_front_view.jpg` |
| 街景点位pid | 点位标识 | 如 `xxx` |
| 安全性评级 | 总体评级(文字) | 很差…很好,未评则为空 |
| 完成状态 | 是否完成 | 已完成 / 未完成 |
| SR1评级 | 自然监视不足 维度评级(文字) | 很差…很好 |
| SR2评级 | 环境失序 维度评级(文字) | 很差…很好 |
| 归因维度 | 勾选的问题维度,`、` 连接 | 如 `自然监视不足、环境失序`;勾选无明显问题时为 `无明显问题` |
| 无明显问题 | 是否勾选 | 是 / (空) |
| 提交时间 | 最后操作时间 | ISO 时间戳 |

---

### 工作表② `问题明细` — 每条问题一行(tidy 长表,提取主用)

**这是后续自动化提取的核心表。** 一条问题 = 一行,无任何分隔符拼接,数量可变天然适配。

| 列名 | 含义 | 外部盲评取值 | 内部扩展取值 |
|---|---|---|---|
| 评价者编号 | 评价者 ID | 1–13 | 14–26 |
| 图像序号 | 分配顺序 | 1–20 | 1–20 |
| 图像文件名 | 原始文件名 | 文件名 | 文件名 |
| 街景点位pid | 点位标识(冗余便于关联) | pid | pid |
| 安全性评级 | 该图总体评级(冗余便于过滤) | 很差…很好 | 很差…很好 |
| 问题序号 | 该图内的问题顺序 | 1,2,3… | 1,2(最多 2) |
| 问题类型 | 问题类别 | **评价者自填,≤10 字** | SR 维度名:`自然监视不足` / `环境失序` |
| 解释说明 | 问题分析文字 | 评价者自填分析文本 | (空,内部仅勾选不填文) |

**可变数量规则(重点):**
- 一张图填了 N 条问题 → 明细表 N 行,问题序号 1…N
- 一张图勾选"无明显问题" → 明细表**该图无行**(状态在总表)
- 一张图未填任何问题(未完成) → 明细表**该图无行**(完成状态=未完成)
- → 明细表行数 = 该评价者所有图的有效问题条目总和,与各图问题数严格对应

> 说明:外部模式下,只有"问题类型"非空的条目才进入明细表;仅填了类型未填解释的条目也会保留(解释说明为空),不会静默丢失。是否"完成"以总表`完成状态`为准(要求类型与解释都填)。

## 2. JSON 结构(程序处理用,可选)

```jsonc
{
  "evaluator_id": 5,                 // 评价者编号
  "mode": "external",                // "external" | "internal"
  "exported_at": "2026-07-14T...",   // 导出时间
  "images": [
    {
      "image_index": 1,              // 图像序号 1-20
      "image_name": "xxx_front_view.jpg",
      "pid": "xxx",
      "level_rating_value": 2,       // 1-5 整数;未评则 null
      "level_rating": "较差",         // 文字;未评则 null
      "completed": true,

      // —— 仅 external 模式 ——
      "no_issue": false,
      "no_issue_explain": "",
      "problems": [                  // 结构化数组,数量可变;no_issue 时为 []
        { "seq": 1, "type": "照明不足", "analysis": "该段步道两侧高绿化遮挡..." },
        { "seq": 2, "type": "视线受阻", "analysis": "..." }
      ],

      // —— 仅 internal 模式 ——
      "sr1_rating_value": 2, "sr1_rating": "较差",
      "sr2_rating_value": 3, "sr2_rating": "一般",
      "issue_selection": ["SR1", "SR2"],      // 或 ["no_issue"]
      "issue_selection_names": ["自然监视不足", "环境失序"],

      "timestamp": "2026-07-14T..."
    }
  ]
}
```

## 3. 回收后提取示例(Python / pandas)

### 3.1 批量加载所有评价者文件

```python
import pandas as pd, glob, json

# 把所有"安全性评价_评价者XX.xlsx"放同一目录
files = sorted(glob.glob("安全性评价_评价者*.xlsx"))

sum_list, prob_list = [], []
for f in files:
    xl = pd.ExcelFile(f)
    sum_list.append(pd.read_excel(xl, "评分总表"))
    prob_list.append(pd.read_excel(xl, "问题明细"))

df_sum  = pd.concat(sum_list,  ignore_index=True)   # 评分总表(全量)
df_prob = pd.concat(prob_list, ignore_index=True)   # 问题明细(全量)
```

### 3.2 按模式拆分

```python
ext_sum  = df_sum [df_sum ["评价者编号"] <= 13]
int_sum  = df_sum [df_sum ["评价者编号"] >= 14]
ext_prob = df_prob[df_prob["评价者编号"] <= 13]
int_prob = df_prob[df_prob["评价者编号"] >= 14]
```

### 3.3 评级数值化 + 每点位共识(每图 5 人)

```python
rating_map = {"很差":1, "较差":2, "一般":3, "较好":4, "很好":5}

ext_num = ext_sum.assign(rating=ext_sum["安全性评级"].map(rating_map))
consensus = (ext_num.groupby("街景点位pid")["rating"]
             .agg(["mean", "std", "count", list]))
# count 应为 5(每图 5 位评价者);可据此核查漏交
```

### 3.4 提取每个点位被指出的问题(核心)

```python
# 外部:某点位所有评价者填写的全部问题(逐条独立行)
pid = "某pid"
view = ext_prob[ext_prob["街景点位pid"] == pid][
    ["评价者编号", "问题序号", "问题类型", "解释说明"]]
print(view.to_string(index=False))

# 统计该点位被提及的问题类型频次(5 人汇总)
freq = (ext_prob[ext_prob["街景点位pid"]==pid]
        .groupby("问题类型").size().sort_values(ascending=False))
```

### 3.5 内部维度勾选汇总

```python
# 每个 pid 的 SR1/SR2 平均评级 + 被勾选为问题的次数
int_num = int_sum.assign(
    sr1=int_sum["SR1评级"].map(rating_map),
    sr2=int_sum["SR2评级"].map(rating_map))
int_consensus = int_num.groupby("街景点位pid").agg(
    sr1_mean=("sr1","mean"), sr2_mean=("sr2","mean"), n=("sr1","count"))

# 各维度被勾选次数(明细表逐行计数)
dim_freq = int_prob.groupby("问题类型").size()
```

### 3.6 用 JSON 做更灵活的结构化处理

```python
data = []
for f in sorted(glob.glob("安全性评价_评价者*.json")):
    data.append(json.load(open(f, encoding="utf-8")))
```

## 4. 字段对应速查(代码 ↔ 导出)

| 代码字段(eval.js) | 导出字段 |
|---|---|
| `evaluatorId` | 评价者编号 |
| 图像在 `S.images` 的索引+1 | 图像序号 |
| `im.name` | 图像文件名 |
| `pidOf(im.name)` | 街景点位pid |
| `r.level_rating` (1–5) | 安全性评级(文字)/JSON 另给数值 |
| `r.attributions[].type` | 问题类型(外部) |
| `r.attributions[].analysis` | 解释说明(外部) |
| `r.no_issue` / `r.no_issue_explain` | 无明显问题 / 无明显问题说明 |
| `r.sr1_rating` / `r.sr2_rating` | SR1评级 / SR2评级(内部) |
| `r.issue_selection` (`SR1`/`SR2`/`no_issue`) | 归因维度(内部) |
| `r.timestamp` | 提交时间 |

## 5. 常见核查清单

- **漏交核查**:某模式下每个 pid 应有 5 份评价;`consensus` 的 `count` 列 < 5 即该图评价人数不足。
- **完成度核查**:`评分总表.完成状态` 含"未完成"的评价者,其数据可能缺评级或缺问题,按需剔除或补交。
- **空值约定**:Excel 中空字段为空字符串;JSON 中评级未评为 `null`、问题数组为 `[]`。
- **模式混淆**:外部问题类型是自由文本(需后续主题归类),内部问题类型是固定维度名(自然监视不足/环境失序),二者不要混在同一统计中。
